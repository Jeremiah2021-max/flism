# Flism — Render Deployment Guide

## Architecture (3 separate Render services)

| Service | Type | URL | Purpose |
|---|---|---|---|
| `flism-app` | Web (Node) | `https://flism-app.onrender.com` | Express API + student web app |
| `flism-admin` | Static Site | `https://flism-admin.onrender.com` | Admin panel |
| Render Postgres | Database | internal | PostgreSQL database |

The API and student app share a single Node service.  
The admin panel is a standalone static site that calls the API at `https://flism-app.onrender.com/api/*`.

---

## Step-by-step setup on Render

### 1. Create a PostgreSQL database

In the Render dashboard → New → PostgreSQL  
- Name: `flism-db`  
- Region: Frankfurt  
- Plan: Free  
Copy the **Internal Database URL** — you'll need it for `DATABASE_URL`.

### 2. Deploy from the render.yaml (Blueprint)

Render can auto-create both services from the `render.yaml` in the repo root:

- Dashboard → New → Blueprint  
- Connect your GitHub repo  
- Render will detect `render.yaml` and create both `flism-app` and `flism-admin`

### 3. Set environment variables for `flism-app`

After the Blueprint creates the services, go to **flism-app → Environment** and set:

| Key | Value |
|---|---|
| `DATABASE_URL` | Paste the Postgres Internal Database URL |
| `PAYSTACK_SECRET_KEY` | Your Paystack live secret key |
| `PAYSTACK_PUBLIC_KEY` | Your Paystack live public key |
| `JWT_SECRET` | Auto-generated by Render — leave as-is |
| `NODE_ENV` | `production` — already set in render.yaml |
| `EXPO_PUBLIC_API_URL` | `https://flism-app.onrender.com` — already set |
| `ADMIN_APP_URL` | `https://flism-admin.onrender.com` — already set |
| `STUDENT_APP_URL` | `https://flism-app.onrender.com` — already set |

### 4. Trigger a deploy

Both services will build automatically on first push. If not, click **Manual Deploy → Deploy latest commit** on each.

### 5. Create the admin account

Once `flism-app` is live, open the Render Shell for `flism-app` and run:

```sh
node server/scripts/create-admin.js admin@yourdomain.com YourPassword123! "Your Name"
```

Or use the auto-seeded default: `admin@flism.com` / `Admin@Flism2024`  
**Change this immediately after first login.**

## Custom domains (optional)

Set custom domains in each service's **Settings → Custom Domains**.  
If you add a custom domain for either service, add it to the `ADMIN_APP_URL` or `STUDENT_APP_URL`  
env var on `flism-app` so CORS keeps working.

## Local development

```sh
# Terminal 1 — API (port 5000)
cd server && npm install && node index.js

# Terminal 2 — Student app (port 3000, proxied through Express at 5000)
cd mobile && npm install && npx expo start --web --port 3000

# Terminal 3 — Admin app (port 3001, standalone)
cd admin-app && npm install && EXPO_PUBLIC_API_URL=http://localhost:5000 npx expo start --web --port 3001
```

The student app is accessed at `http://localhost:5000` (Express proxies to port 3000).  
The admin app is accessed directly at `http://localhost:3001`.
