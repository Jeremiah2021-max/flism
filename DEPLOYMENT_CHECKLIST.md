## Deployment Steps

### 1. Create PostgreSQL Database
1. Go to Render Dashboard â†’ New â†’ PostgreSQL
2. Name: `flism-db`
3. Region: Frankfurt
4. Plan: Free
5. Copy the **Internal Database URL**

### 2. Deploy from Blueprint
1. Go to Render Dashboard â†’ New â†’ Blueprint
2. Connect your GitHub repository
3. Render will detect `render.yaml` and create both services
4. Wait for initial deployment to complete

### 3. Configure Environment Variables
After Blueprint deployment, go to **flism-app â†’ Environment** and set:

| Variable | Value | Source |
|----------|-------|--------|
| `DATABASE_URL` | Paste Postgres Internal URL | Render Dashboard |
| `PAYSTACK_SECRET_KEY` | Your Paystack secret | Paystack Dashboard |
| `PAYSTACK_PUBLIC_KEY` | Your Paystack public key | Paystack Dashboard |
| `ADMIN_EMAIL` | admin@yourdomain.com | Your choice |
| `ADMIN_PASSWORD` | Strong password | Your choice |
| `ADMIN_NAME` | Admin display name | Your choice |

**Note:** These are marked `sync: false` in render.yaml, so they must be set manually.

### 4. Trigger Redeploy
After setting environment variables:
1. Go to **flism-app** â†’ Manual Deploy â†’ Deploy latest commit
2. Go to **flism-admin** â†’ Manual Deploy â†’ Deploy latest commit

### 5. Verify Deployment
1. Check health endpoint: `https://flism-app.onrender.com/api/health`
2. Should return:
   ```json
   {
     "status": "ok",
     "app": "Flism API",
     "env": "production",
     "timestamp": "...",
     "database": "connected",
     "uptime": ...
   }
   ```

### 6. Create Admin Account
**Option 1: Auto-seed (Recommended)**
- If you set `ADMIN_EMAIL`, `ADMIN_PASSWORD`, and `ADMIN_NAME` in step 3, the admin account will be created automatically.

**Option 2: Manual Creation**
1. Open Render Shell for `flism-app`
2. Run: `node server/scripts/create-admin.js admin@yourdomain.com YourPassword123! "Your Name"`

### 7. Test Applications
- Student app: `https://flism-app.onrender.com`
- Admin panel: `https://flism-admin.onrender.com`
- API health: `https://flism-app.onrender.com/api/health`

### Known Limitations
1. **File Uploads**: Render's filesystem is ephemeral. Uploaded files will be lost on redeploy. Consider cloud storage (AWS S3, Cloudflare R2) for production.
2. **Free Tier**: Render free tier spins down after 15 minutes of inactivity. First request may take 30-60 seconds.

## Troubleshooting

### Database Connection Failed
- Verify `DATABASE_URL` is correct
- Check PostgreSQL database is running
- Ensure database is in the same region (Frankfurt)

### Admin Account Not Created
- Check logs for admin seed errors
- Verify `ADMIN_EMAIL`, `ADMIN_PASSWORD`, `ADMIN_NAME` are set
- Try manual creation via shell

### Migrations Failed
- Check logs for migration errors
- Run manually: `cd server && npm run migrate`
- Verify database connection

### CORS Errors
- Verify `ADMIN_APP_URL` and `STUDENT_APP_URL` are set correctly
- Check custom domains if used
- Ensure allowed origins match your URLs

## Rollback Plan

If deployment fails:
1. Revert code changes via git
2. Redeploy previous commit
3. Rollback migrations if needed: `npm run migrate:down`
